# SoftFloat Golden Oracle

This directory contains local oracle tooling for validating the pure MoonBit
implementation against Berkeley SoftFloat Release 3e.

Build both specialization binaries:

```bash
tools/build.sh
```

Regenerate the core v1 fixture set, its matching MoonBit whitebox test, and the
batched fixture tests:

```bash
tools/test_gen.mjs core
moon fmt
```

Regenerate the full sweep fixture set, its compact MoonBit digest test, and the
batched fixture tests:

```bash
tools/test_gen.mjs sweep
moon fmt
```

Replay all committed vectors against the freshly built binaries:

```bash
tools/check_vectors.sh
```

`check_smoke.sh` is kept as a compatibility alias:

```bash
tools/check_smoke.sh
```

The build emits:

- `tools/oracle-8086-sse`
- `tools/oracle-riscv`

The RISCV binary is a host executable using the RISCV SoftFloat
specialization, not a RISC-V target binary.

## CLI

```bash
tools/oracle-8086-sse <operation> <rounding> <tininess> <exact> <hex-input>...
tools/oracle-8086-sse --batch <operation> <rounding> <tininess> <exact> < input-lines
```

Fields:

- `operation`: MoonBit snake-case v1 operation name, such as `f64_add` or
  `f32_to_ui32_min_mag`.
- `rounding`: `near_even`, `min_mag`, `min`, `max`, `near_max_mag`, or `odd`.
- `tininess`: `before` or `after`.
- `exact`: `0` or `1`; ignored by operations that do not take SoftFloat's
  operation-local exact flag.
- `hex-input`: fixed-width bit pattern or two's-complement integer input.

Output is one line:

```text
specialization operation rounding tininess exact inputs... output flags
```

Float outputs are fixed-width hex bit patterns. Integer outputs are fixed-width
two's-complement hex values so specialization-specific invalid constants can be
compared without host-width ambiguity. Boolean outputs are `0` or `1`. Flags are
two-digit hex SoftFloat exception bits: `01` inexact, `02` underflow, `04`
overflow, `08` infinite, `10` invalid.

Example:

```bash
tools/oracle-8086-sse f64_add near_even after 1 3FF0000000000000 3FF0000000000000
```

Expected record:

```text
x86_8086_sse f64_add near_even after 1 3FF0000000000000 3FF0000000000000 4000000000000000 00
```

Batch mode reads one whitespace-separated input tuple per line from stdin and
emits the same record format. `tools/test_gen.mjs check` uses this mode so large
fixture files can be replayed without starting one oracle process per record.

## Fixtures

Fixtures live under `tools/vectors/`. They use the same line-oriented
record format emitted by the CLI. Generated sweep fixtures use a deterministic
64-bit LCG seeded with `0x5F0F7A7C` and are grouped by operation and
specialization rather than merged into one large file.

`vectors/smoke.txt` contains a small committed sanity set for both
specializations. `vectors/core.txt` is generated by
`tools/test_gen.mjs core`; it contains one representative case for every v1
function-style operation and both specializations, and
`golden_core_wbtest.mbt` checks the MoonBit implementation against those
committed records.

`tools/test_gen.mjs sweep` creates the larger committed fixture set:

- `edge_*`: boundary values for every included format, all arithmetic modes
  `near_even`, `min`, `max`, and `odd`, both tininess modes, all six
  round-to-int and integer-conversion modes where relevant, and exact/inexact
  flag settings for conversion APIs that take `exact`.
- `random_*`: fixed-seed random arithmetic records. Binary F16 operations use
  65,536 sampled pairs per operation and specialization; binary F32 and F64
  operations use 10,000 pairs; F16/F32/F64 FMA use 10,000 triples; F32-to-BF16
  uses 10,000 fixed-seed F32 inputs.
- `exhaustive_bf16_*`: all 65,536 BF16 inputs for signaling-NaN
  classification and BF16-to-F32 conversion.
- `exhaustive_f16_*`: all 65,536 F16 inputs for signaling-NaN classification,
  F16-to-F32/F64 conversion, and selected round-to-int modes.

`golden_sweep_digest_wbtest.mbt` is generated with the sweep fixtures. It
recreates the same deterministic input rows in MoonBit, folds each result and
exception flag into a per-file digest, and checks those digests with
`@debug.assert_eq` against the C-oracle records.

`tools/test_gen.mjs packages` converts the committed fixtures under
`vectors/` into MoonBit test packages under `sweep_test/batch<N>/`. The core
and smoke files are emitted as direct record assertions; sweep files are emitted
as 200-record digest range tests. Generated batch packages use at most 10
MoonBit test files per package and at most 100 `test` blocks per file.

The default generated `vectors/` directory is about 121 MiB and currently
contains 2,219,816 oracle records. For tooling checks without regenerating the
full set, use:

```bash
tools/test_gen.mjs sweep --quick --skip-edge
```
